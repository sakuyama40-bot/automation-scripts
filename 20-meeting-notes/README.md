# 20-meeting-notes — 議事録・タスクメモ Markdown 整形ツール

生のメモを貼り付けると Claude が構造化された Markdown に整形して自動保存。

## セットアップ

```bash
cp .env.example .env
# .env に ANTHROPIC_API_KEY を設定
```

## 使い方

```bash
node notes-formatter.js
```

## 2つのモード

### 議事録モード

生の会議メモを以下の構成に整理します：

```markdown
# 議事録 - 2026年06月27日（土）
## automation-scripts 進捗MTG

## 出席者
- 作山 哲
- 田中

## 議題
1. automation-scripts の進捗確認
2. 次回スケジュール

## 議事内容
...

## 決定事項
- [x] 01〜12 のツールを今週中に完成

## ネクストアクション
- [ ] 20番ツールの実装（担当: 作山、期限: 6/30）

## 次回予定
- 来週金曜 定例MTG
```

### タスクメモモード

バラバラなメモを優先度別タスクリストに整理します：

```markdown
# タスクリスト - 2026年06月27日（土）

## 🔴 高優先度
- [ ] commitlint の設定確認

## 🟡 中優先度
- [ ] README 更新

## 🟢 低優先度 / 保留
- [ ] ドキュメント整備
```

## 入力方法

| 選択 | 内容 |
|---|---|
| 直接入力 | ターミナルに貼り付け。`---` 単独行または空行2回で終了 |
| ファイル読み込み | `.txt` や `.md` ファイルのパスを指定 |

## 保存先

`outputs/YYYY-MM-DD_meeting.md` または `outputs/YYYY-MM-DD_tasks.md`

同日に複数回実行した場合は `_2.md`, `_3.md` と自動で番号が付きます。

## APIキーなしでも動く

テンプレートを生成して生メモをそのまま埋め込みます。後で手動整形できる状態で保存されます。

## モデル

`claude-sonnet-4-6` を使用（18-daily-report の haiku より高品質な整形のため）。

## 依存パッケージ

なし（Node.js 標準モジュールのみ）
